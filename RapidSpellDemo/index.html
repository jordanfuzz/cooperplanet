RapidSpell Web .NET Evaluation Examples

Welcome to the RapidSpell Web .NET Evaluation. Below are links to example uses of this spell checker control which demonstrate how easy it is to apply your own style, layout and text to the various elements.

Please note that this is an evaluation with limited functionality (no user dictionary functionality) and a dictionary without any words beginning with 'z'.

To view these examples the Keyoti.RapidSpellWebEval.dll and Keyoti.RapidSpellMDict.dll files must both be in your GAC or in the bin directory of your web application.

Using RapidSpell Web is as easy as adding
<RapidSpellWeb:RapidSpellWebLauncher id="rapidSpellWebLauncher" runat="server" TextComponentName="myForm.sourceTextBox" CallBack="myForm.sourceTextBox" Mode="popup" RapidSpellWebPage="RapidSpellCheckerPopUp.aspx"/> to your code

Remember, the pages and spell checker will load slowly the first time as they are compiled.

exampleTextBox1.aspx - Styled according to the Keyoti web site, the spell checker used in pop up mode.

exampleTextBox2-Simple.aspx - Default, no specific layout.

exampleTextBox4-Custom.aspx - Custom layout in the spell checker box.

exampleTextBox5-Separate.aspx - Keyoti style, but using the spell checker in 'separate' mode.

The following 2 examples require RichTextBox.com's RichTextBox, you should place the RichTextBox DLL in your bin directory.

exampleTextBox6-3rdParty.aspx - RapidSpell Web running with RichTextBox.

exampleTextBox7-3rdParty-Separate.aspx - RapidSpell Web running with RichTextBox (separate mode).

Some advanced uses.

exampleTextBox8-2boxes.aspx - RapidSpell Web running with 2 text boxes.

exampleTextBox9-CloseEvent.aspx - Capturing the client side 'spelling finished' event.

exampleTextBox13-Image.aspx - Using images instead of a button.

exampleTextBox15-NonEnglishUI.aspx - Setting the UI and dictionary at run-time to a different language.
